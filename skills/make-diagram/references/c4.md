# Diagramas C4 (diagrams.c4)

O [modelo C4](https://c4model.com/) descreve arquitetura de software em níveis de
zoom: **Context** (sistemas e pessoas), **Container** (aplicações/serviços dentro de
um sistema), **Component** e **Code**. Com a lib `diagrams`, os níveis Context e
Container são os mais práticos — use os mesmos nós variando o nível de detalhe.

## Classes disponíveis

```python
from diagrams.c4 import Person, Container, Database, System, SystemBoundary, Relationship
```

| Classe | Uso | Parâmetros principais |
|---|---|---|
| `Person` | Ator/usuário | `name`, `description`, `external=True/False` |
| `System` | Sistema de software (nível Context) | `name`, `description`, `external=True/False` |
| `Container` | Aplicação/serviço dentro do sistema | `name`, `technology`, `description` |
| `Database` | Armazenamento de dados | `name`, `technology`, `description` |
| `SystemBoundary` | Fronteira do sistema (context manager, como `Cluster`) | `name` |
| `Relationship` | Aresta com rótulo descritivo | texto (ex.: `"Makes API calls to [JSON/HTTPS]"`) |

Marque sistemas/pessoas fora do seu controle com `external=True` (renderizados em
cinza, convenção C4).

## Tamanho dos nós (evita texto transbordando)

Os nós C4 têm tamanho **fixo** (`fixedsize=true`, 2.6×1.6 pol.), mas a `description`
é quebrada em linhas de até 40 caracteres — linhas nesse limite **não cabem** na
caixa padrão e o texto transborda/corta no PNG. Sempre que usar `description`,
aumente a caixa passando `width`/`height` (os kwargs vão direto pro Graphviz):

```python
C4_SIZE = {"width": "3.9", "height": "2.1"}  # cabe 3 linhas de 40 chars

api = Container(name="API", technology="Go", description="...", **C4_SIZE)
```

Aplique o mesmo `**C4_SIZE` em todos os nós com descrição (Person, System,
Container, Database) — além de caber, os nós ficam uniformes. Descrições também
são truncadas em 3 linhas (com `...`); mantenha-as com até ~120 caracteres.

## Fila/broker de mensageria (cilindro horizontal)

Na notação C4, filas/brokers são desenhados como **cilindro deitado** ("pipe"),
mas o Graphviz não tem esse shape (o `cylinder` é só vertical e não aceita
rotação). A solução da skill: usar uma imagem de cilindro horizontal como fundo
do nó — o rótulo C4 é desenhado por cima da imagem.

1. Copie `assets/c4_queue.png` (da pasta desta skill) para o diretório do
   arquivo do diagrama, assim o `.py` continua regenerável sozinho.
2. Adicione este helper ao diagrama:

```python
import os
from diagrams.c4 import C4Node

def Queue(name, technology="", description="", **kwargs):
    attrs = {
        "name": name, "technology": technology, "description": description,
        "type": "Queue",
        "shape": "none",
        "style": "",
        "fillcolor": "transparent",
        "image": os.path.join(os.path.dirname(os.path.abspath(__file__)), "c4_queue.png"),
        "imagescale": "true",
    }
    attrs.update(C4_SIZE)
    attrs.update(kwargs)
    return C4Node(**attrs)

broker = Queue(name="Broker de Mensagens", technology="RabbitMQ",
               description="Distribui eventos para os consumidores.")
```

O rótulo sai como `[Queue: RabbitMQ]` e o nó se conecta com `Relationship`
normalmente. A imagem já tem a proporção de `C4_SIZE` (3.9×2.1) e a mesma cor
dos Containers.

## Pessoa/ator com ícone (`assets/c4_user.png`)

Por padrão, `Person` renderiza uma caixa azul sólida (sem ícone). A skill tem uma
imagem alternativa (`assets/c4_user.png`, um ícone de usuário) para usar no lugar
dessa caixa quando o diagrama pedir um ator mais reconhecível visualmente.

**Importante:** essa imagem tem proporção própria (~1.11, quase quadrada — mais
larga que alta, mas bem menos que a paisagem de `C4_SIZE` (3.9×2.1, ~1.86)). Para
não esticar/achatar o ícone, use uma caixa com essa mesma proporção em vez de
`C4_SIZE`, mantendo a mesma altura dos demais elementos (Container/System/Database)
para alinhar na mesma linha/coluna do diagrama:

```python
PERSON_SIZE = {"width": "2.33", "height": "2.1"}  # mesma altura de C4_SIZE, largura ajustada à proporção do ícone (804x724px)
```

1. Copie `assets/c4_user.png` (da pasta desta skill) para o diretório do arquivo
   do diagrama, assim o `.py` continua regenerável sozinho (mesmo tratamento do
   `c4_queue.png`).
2. Adicione este helper ao diagrama:

```python
import os
from diagrams.c4 import C4Node

def User(name, description="", external=False, **kwargs):
    attrs = {
        "name": name, "description": description,
        "type": "External Person" if external else "Person",
        "shape": "none",
        "style": "",
        "fillcolor": "transparent",
        "image": os.path.join(os.path.dirname(os.path.abspath(__file__)), "c4_user.png"),
        "imagescale": "true",
    }
    attrs.update(PERSON_SIZE)
    attrs.update(kwargs)
    return C4Node(**attrs)

customer = User(name="Cliente", description="Cliente do banco com contas pessoais.")
```

Se a imagem do ícone for trocada novamente, recalcule `PERSON_SIZE` a partir da
nova proporção (`largura_px / altura_px * altura_desejada`) para não distorcer.

Use `User(...)` sempre que o nó representar uma pessoa/ator no diagrama — ele
substitui `Person(...)` mantendo os mesmos parâmetros (`name`, `description`,
`external`). Se o diagrama não precisar do ícone (ex.: pedido explícito do
usuário por uma caixa simples), `Person(...)` do `diagrams.c4` continua
disponível normalmente.

## Convenções que fazem um bom C4

- Rótulos de relacionamento descrevem **a ação e o protocolo**: `>> Relationship("Consulta saldo via [JSON/HTTPS]") >>`
- Em `Container`, sempre preencha `technology` ("Java e Spring MVC", "Node.js")
- Prefira `direction="TB"` e `graph_attr={"splines": "spline"}` — o layout C4 fica mais legível
- Uma `SystemBoundary` por sistema; o que está fora dela são atores e sistemas externos

## Exemplo completo (Container diagram)

```python
from diagrams import Diagram
from diagrams.c4 import Person, Container, Database, System, SystemBoundary, Relationship

graph_attr = {"splines": "spline"}
C4_SIZE = {"width": "3.9", "height": "2.1"}

with Diagram("Container diagram - Internet Banking", direction="TB",
             graph_attr=graph_attr, show=False):
    customer = Person(name="Cliente", description="Cliente do banco com contas pessoais.",
                      **C4_SIZE)

    with SystemBoundary("Internet Banking"):
        spa = Container(name="Single-Page App", technology="JavaScript e Angular",
                        description="Funcionalidades de banking no navegador.", **C4_SIZE)
        api = Container(name="API", technology="Java e Spring MVC",
                        description="Funcionalidades via API JSON/HTTPS.", **C4_SIZE)
        database = Database(name="Database", technology="Oracle",
                            description="Cadastros, credenciais, logs de acesso.", **C4_SIZE)

    email = System(name="Sistema de E-mail", description="Exchange interno.",
                   external=True, **C4_SIZE)
    mainframe = System(name="Mainframe", description="Core bancário.",
                       external=True, **C4_SIZE)

    customer >> Relationship("Usa [HTTPS]") >> spa
    spa >> Relationship("Chama API [JSON/HTTPS]") >> api
    api >> Relationship("Lê e escreve em") >> database
    api >> Relationship("Envia e-mail via [SMTP]") >> email
    api >> Relationship("Chama [XML/HTTPS]") >> mainframe
    customer << Relationship("Recebe e-mails de") << email
```
